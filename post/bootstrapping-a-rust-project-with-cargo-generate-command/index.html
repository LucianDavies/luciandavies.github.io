Bootstrapping a rust project with cargo generate command

June 12, 2020

Intro

Many times as with the case with trying to learn something new you find small obstacles that slow down your progress and before you know it you have abandoned your shiny new adventure. This is a common problem in software development that we have CLI’s and generators design to catapult you into the exciting bits. As much as there is a need to learn the basic process of building your project sometimes you want to just try something and see if its worth chasing. Today I want to introduce you to cargo generate. This command line tool generates rust projects based on community provided templates.

Basic dependencies

To use cargo generate, we need to set up some tools listed below.

  • Rust & Cargo
  • The GNU Embedded Toolchain for Arm & Openocd
  • Cargo generate

Because rust has such good documentation I wont reiterate the actual steps but the importance of the tools highlighted above is that the program we will generate needs an already setup environment to build and run.

Using cargo generate

The program itself is nothing special but excited me when I stumbled onto this project was the various templates I could build my project from. At the time of writing we had the following

I for my project chose the bluepill project and it was as easy as searching the list for something compatible clicking on the link to obtain the github link and running the code below in my terminal

cargo generate --git https://github.com/mendelt/bluepill-template

Voilà

Again I cant stress how sometimes bootstrapping a project is the hardest part and for me this tool is a time saver. I will try some of the other templates like the cmdr or the CLI but for now let me explore my bluepill and see what interesting things I can learn.
